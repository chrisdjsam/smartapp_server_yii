getRelatedData (line 187)

Finds the related primary keys specified in the form POST.

Only for HAS_MANY and MANY_MANY relations.

  • return: An array where the keys are the relation names (string) and the values are arrays with the related model primary keys (int|string) or composite primary keys (array with pk name (string) => pk value (int|string)). Example of returned data: 
     array(
   'categories' => array(1, 4),
   'tags' => array(array('id1' => 3, 'id2' => 7), array('id1' => 2, 'id2' => 0)) // composite pks
 )
    An empty array is returned in case there is no related pk data from the post. This data comes directly from the form POST data.
  • see: GxHtml::activeCheckBoxList
  • throws: InvalidArgumentException If uncheckValue is null.
  • access: protected
array getRelatedData (array $form, array $relations, [string $uncheckValue = ''])
  • array $form: The post data.
  • array $relations: A list of model relations in the format returned by CActiveRecord::relations.
  • string $uncheckValue: Since Yii 1.1.7, htmlOptions (in CHtml::activeCheckBoxList) has an option named 'uncheckValue'. If you set it to different values than the default value (''), you will need to set the appropriate value to this argument. This method can't be used when 'uncheckValue' is null.
loadModel (line 81)

Returns the data model based on the primary key or another attribute.

This method is designed to work with the values passed via GET. If the data model is not found or there's a malformed key, an HTTP exception will be raised. #MethodTracker This method is based on the gii generated method controller::loadModel, from version 1.1.7 (r3135). Changes:

  • Support to composite PK.
  • Support to use any attribute (column) name besides the PK.
  • Support to multiple attributes.
  • Automatically detects the PK column names.

  • return: The loaded model.
  • see: GxActiveRecord::pkSeparator
  • throws: CHttpException if there's an invalid request (with code 400) or if the model is not found (with code 404).
  • access: public
GxActiveRecord loadModel (mixed $key, string $modelClass)
  • mixed $key: The key or keys of the model to be loaded. If the key is a string or an integer, the method will use the tables' PK if the PK has a single column. If the table has a composite PK and the key has a separator (see below), the method will detect it and use it. 
     $key = '12-27'; // PK values with separator for tables with composite PK.
    If $key is an array, it can be indexed by integers or by attribute (column) names, as for CActiveRecord::findByAttributes. If the array doesn't have attribute names, as below, the method will use the table composite PK. 
     $key = array(
   12,
   27,
   ...,
 );
    If the array is indexed by attribute names, as below, the method will use the attribute names to search for and load the model. 
     $key = array(
   'model_id' => 44,
      ...,
 );
    Warning: each attribute used should have an index on the database and the set of attributes used should identify only one item on the database (the attributes being ideally part of or multiple unique keys).
  • string $modelClass: The model class name.
