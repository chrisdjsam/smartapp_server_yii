static method saveMultiple (line 549)

Saves multiple active records.

This method can detect automatically all new active records having a BELONGS_TO relation (to HAS_ONE or to HAS_MANY) and fill in the data for their FK if it is null. The order of the active records in the $models array parameter is important to make it work. The models that need to be saved first should come first in the array.

  • return: Whether the saving succeeds.
  • see: CActiveRecord::save
  • see: GxActiveRecord::saveWithRelated()
  • throws: Exception If "detectRelations" is true and the related model is not found.
  • access: public

getRelationLabel (line 77)

Returns the text label for the specified active record relation, attribute or class property.

The labels are the user friendly names displayed in the views. If defined in the model, the label for its attribute, property or relation is returned. If not defined in the model (in CModel::attributeLabels), the label is generated using the related active record class label (via GxActiveRecord::label) (for FK attributes and relations) or using CModel::generateAttributeLabel (for other attributes and class properties).

  • return: The label.
  • see: GxActiveRecord::label()
  • throws: InvalidArgumentException If an attribute name is found and is not the last item in the relationName parameter.
  • access: public
string getRelationLabel (string $relationName, [integer $n = null], [boolean $useRelationLabel = true])
  • string $relationName: The relation, attribute or class property name. This method supports chained relations in the form of "post.author.name".
  • integer $n: The number value. This is used to support plurals. In the default implementation, when this argument is null, if the relation is BELONGS_TO or HAS_ONE, the singular form is returned. If the relation is HAS_MANY or MANY_MANY, the plural form is returned. If this argument is null and the relation is not one of the types listed above, the singular form is returned. For most languages, 1 means singular and all other values mean plural. Defaults to null. Note: It is not supported when returning labels for attributes or class properties.
  • boolean $useRelationLabel: Whether to use the relation label for the FK attribute. When true, if the specified attribute name is a FK, the corresponding related AR label will be used. Defaults to true. Note: this will only work when there is no label defined in CModel::attributeLabels for this attribute.

saveWithRelated (line 340)

Saves the current record and its MANY_MANY relations.

This method will save the active record and update the necessary pivot tables for the MANY_MANY relations. The pivot table is the table that maps the relationship between two other tables in a MANY_MANY relation. This method won't save data on other active record models.

  • return: Whether the saving succeeds.
  • see: GxActiveRecord::pivotModels()
  • access: public
boolean saveWithRelated (array $relatedData, [boolean $runValidation = true], [array $attributes = null], [array $options = array()])
  • array $relatedData: The relation data in the format returned by GxController::getRelatedData.
  • boolean $runValidation: Whether to perform validation before saving the record. If the validation fails, the record will not be saved to database. This applies to all (including related) models. This does not apply for related models when in batch mode. This does not apply for deletes.
  • array $attributes: List of attributes that need to be saved. Defaults to null, meaning all attributes that are loaded from DB will be saved. This applies only to the main model.
  • array $options: Additional options. Valid options are:
    • 'withTransaction', boolean: Whether to use a transaction.
    • 'batch', boolean: Whether to try to do the deletes and inserts in batch. While batches may be faster, using active record instances provides better control, validation, event support etc. Batch is only supported for deletes.
